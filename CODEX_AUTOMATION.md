## 任务一：更新情报（skill.md 第六节）

### 步骤

1. **联网搜索**未来 48 小时内有比赛的球队最新消息：
   - 伤停（确认/存疑/赛事报销）
   - 官方首发名单（如已公布）
   - 热身赛结果、状态评估
   - 搜索关键词示例：`"World Cup 2026" injuries lineup [team name] [date]`

2. **只重写** `skill.md` 文件末尾的 `## 六、最新情报（每日更新区）` 这一节。  
   **严禁改动第一~五节的任何内容**（那是预测规则和球队资料库）。

3. 第六节格式要求：
```markdown
## 六、最新情报（每日更新区）

> 本节由每日情报流程覆盖更新。**当本节与第四节冲突时，以本节为准**（本节更新）。

**情报日期：2026-06-XX（第X天）**

### 已赛比分（昨日）

| 对阵 | 比分 | 进球 | 备注 |
|------|------|------|------|
| 🇲🇽 墨西哥 vs 🇿🇦 南非 | **2-0** | 进球者+时间 | 红黄牌、来源 |

### 今日比赛（当地时间）

| 对阵 | 场地 | 组别 | 当地开球 |
|------|------|------|---------|
| 队A vs 队B | 城市·球场 | X组 | HH:MM TZ |

### 明日比赛（当地时间）

| 对阵 | 场地 | 组别 |
|------|------|------|
| 队A vs 队B | 城市·球场 | X组 |

### 伤停更新（经确认·来源标注）

**X组（今日·队A vs 队B）：**
- 🏴 球队：球员名（外文名）伤情描述（来源：媒体名）

### 其他重要动态

- 🏴 球队：动态要点
```

4. **质量要求：**
   - 新闻必须真实，优先官方来源（足协/俱乐部公告 > ESPN/BBC Sport > 一般体育媒体）
   - 不确定的标注"存疑"
   - 禁止编造比分、虚构伤病
   - 每条伤停注明来源

5. **上传到服务器：**
```bash
scp skill.md root@8.218.32.133:/opt/worldcup-ai/skill.md
```
   服务端热加载，无需重启。

## 任务二：录入昨日真实比分

### 步骤

1. **联网搜索**昨天所有世界杯比赛的最终比分（搜索 `"World Cup 2026 results [yesterday's date]"`）

2. 对每一场比赛，通过 SSH 在服务器本地执行（避免编码问题）：
```bash
ssh root@8.218.32.133 "curl -s -X POST http://localhost:8080/api/result \
  -H 'Content-Type: application/json' \
  -d '{\"teamA\":\"主队中文名\",\"teamB\":\"客队中文名\",\"stage\":\"小组赛\",\"score\":\"X-X\",\"token\":\"$ADMIN_TOKEN\"}'"
```

3. **响应处理：**
   - `{"ok":true,...}` → 录入成功
   - `{"error":"该对阵没有已锁定的预测，无需录入"}` → 正常，跳过
   - 其他错误 → 记录日志，不中断

4. 全部录完后验证：
```bash
curl -s https://worldcup.youliaoyun.com/api/stats
```
   确认 `total` 和 `outcomeHits` 字段有变化。

### 注意事项
- `teamA` 和 `teamB` 必须用上面48队名单里的**精确中文名**
- `score` 格式：`"2-1"`（主队比分在前，与 teamA 对应）
- `stage` 值：小组赛阶段统一填 `"小组赛"`，淘汰赛填 `"16强"` / `"8强"` / `"半决赛"` / `"决赛"`

## 任务三：更新今日焦点

### 步骤

1. **确定今明两天赛程**，从中挑选 **4 场最值得看的比赛**（优先：强强对话 > 东道主 > 关键晋级战）

2. 通过 SSH 在服务器本地执行：
```bash
ssh root@8.218.32.133 "curl -s -X POST http://localhost:8080/api/fixtures \
  -H 'Content-Type: application/json' \
  -d '{\"token\":\"$ADMIN_TOKEN\",\"fixtures\":[
    {\"a\":\"队A\",\"b\":\"队B\",\"meta\":\"X组 · 北京时间 6/XX HH:00\",\"live\":true},
    {\"a\":\"队C\",\"b\":\"队D\",\"meta\":\"X组 · 北京时间 6/XX HH:00\",\"live\":true},
    {\"a\":\"队E\",\"b\":\"队F\",\"meta\":\"X组 · 北京时间 6/XX HH:00\",\"live\":false},
    {\"a\":\"队G\",\"b\":\"队H\",\"meta\":\"X组 · 北京时间 6/XX HH:00\",\"live\":false}
  ]}'"
```

3. **字段说明：**
   - `a` / `b`：队名，必须是48队名单里的精确中文名
   - `meta`：格式 `"X组 · 北京时间 6/XX HH:00"`（最长40字符）
   - `live: true`：今天要踢的比赛（前端显示红色 TODAY 标签）
   - `live: false`：明天的比赛

4. **时区转换参考：**
   - 美东 ET (UTC-4) → 北京 = +12小时
   - 太平洋 PT (UTC-7) → 北京 = +15小时
   - 墨西哥城 CST (UTC-6) → 北京 = +14小时
   - "当地19:00 ET" = "北京时间次日07:00"

5. 验证：
```bash
curl -s https://worldcup.youliaoyun.com/api/fixtures
```
   确认返回的 JSON 包含刚设置的4场比赛。

## 执行顺序与错误处理

```
1. 搜索新闻 → 更新 skill.md → SCP上传
2. 搜索昨日比分 → 逐场 POST /api/result
3. 确定赛程 → POST /api/fixtures
4. 输出执行报告
```

- 任何一步失败不影响其他步骤继续执行
- 如果联网搜索无结果（如无比赛日），该步骤跳过并在报告中说明
- 世界杯结束后（2026-07-20 以后）自动化应停止执行

## SSH 连接说明

服务器 `8.218.32.133` 使用 SSH key 认证（无密码）。Codex 需要：
1. 将 SSH 私钥配置为 Secret（如 `SSH_PRIVATE_KEY`）
2. 运行时写入 `~/.ssh/id_rsa` 并设置权限 600
3. 将 `8.218.32.133` 加入 known_hosts（或使用 `-o StrictHostKeyChecking=no`）

示例初始化脚本：
```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "$SSH_PRIVATE_KEY" > ~/.ssh/id_rsa && chmod 600 ~/.ssh/id_rsa
ssh-keyscan -H 8.218.32.133 >> ~/.ssh/known_hosts 2>/dev/null
```
